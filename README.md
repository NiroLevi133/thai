## הרצה

```bash
npm install
npm run dev
```

ואז פתח **http://localhost:5173**

הפקודה מריצה שני תהליכים: שרת הנתונים (פורט 5174) ו-Vite (פורט 5173).

## איפה הנתונים

הכל יושב ב-`data/trip.json` — **זה מקור האמת**. כל שינוי באפליקציה נשמר לשם אוטומטית
תוך פחות משנייה (המחוון בראש המסך מראה "נשמר"). לפני כל כתיבה נשמר גיבוי ב-`data/trip.bak.json`.

### ייבוא מחדש מהאקסל

```bash
npm run import     # דורס את data/trip.json מהקובץ המקורי
```

⚠️ זה **מוחק כל שינוי שעשית באפליקציה**. השתמש בזה רק אם אתה רוצה להתחיל מחדש מהאקסל.
הסקריפט מאמת את עצמו ונכשל אם המספרים לא מסתדרים (10 יעדים, 36 לילות, 8 הזמנות, 14,263 ₪).

## המסכים

| מסך | מה יש בו |
|---|---|
| **דשבורד** | ספירה לאחור, התראות ממויינות לפי דחיפות, סקירת כל היעדים |
| **מסלול** | תצוגת יעדים עם רצועת לילות (ירוק=מכוסה, אדום=פתוח), ותצוגת יום-יום |
| **מלונות** | סגורים / מועמדים / נפסלו. סימון "שולם", עריכת תאריכים ומחירים, "סגרתי" למועמד |
| **תחבורה** | כל המעברים בין היעדים + טיסות. סוג, מחיר, משך, אסמכתא |
| **אטרקציות** | לפי יעד, עם שיבוץ ליום ספציפי שמופיע מיד בטיימליין |
| **תקציב** | שולם / מחויב / אומדן, גרפים, והוצאות נוספות |
| **הגדרות** | שער באהט, תקציב, מצב כהה/בהיר, וייצוא |

## התראות

הדשבורד מדרג לפי דחיפות:
- 🔴 **אדום** — דדליין ביטול חינם בתוך 7 ימים
- 🟠 **כתום** — בתוך 21 יום, לילות ללא מלון, סתירות בנתונים
- 🔵 **כחול** — הזמנות שלא סומנו כשולמו, תחבורה לא סגורה

## ייצוא

מתוך מסך ההגדרות:
- **אקסל** — גיבוי בשני גיליונות, בפורמט של הקובץ המקורי
- **.ics** — 35 אירועים ליומן: צ׳ק-אין, צ׳ק-אאוט, כל דדליין ביטול, ומעברי תחבורה
- **JSON** — הנתונים הגולמיים
